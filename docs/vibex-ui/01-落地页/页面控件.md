# 落地页 - 页面控件说明

## 1. 页面级说明

### 1.1 页面骨架结构

```
┌─────────────────────────────────────────┐
│ Header (固定顶部 64px)                  │
│ [Logo] [导航链接] [登录] [开始创建]      │
├─────────────────────────────────────────┤
│ Hero 区域 (100vh - 64px)               │
│ [主标题] [副标题] [CTA按钮]              │
├─────────────────────────────────────────┤
│ 特性展示 Section (4列卡片)              │
├─────────────────────────────────────────┤
│ 使用流程 Section (3步)                  │
├─────────────────────────────────────────┤
│ Footer (链接 + 版权)                    │
└─────────────────────────────────────────┘
```

### 1.2 响应式断点

| 断点 | 宽度 | 布局变化 |
|------|------|----------|
| Desktop | ≥1200px | 4列特性 / 3步流程 |
| Tablet | 768-1199px | 2列特性 / 垂直流程 |
| Mobile | <768px | 1列特性 / 汉堡菜单 |

## 2. 控件级说明

### 2.1 Header 组件

| 控件 | 类型 | 交互 | 数据 |
|------|------|------|------|
| Logo | 文字/图片 | 点击 → 首页 | 固定: "VibeX" |
| 导航链接 | 链接 | hover: 颜色变化 | 固定链接 |
| 登录按钮 | 按钮 | click → 登录页 | - |
| 开始创建按钮 | 主按钮 | hover: 上浮+阴影 | - |

**样式代码**: 见 index.html `<style>` 部分

### 2.2 Hero 区域

| 控件 | 类型 | 交互 | 数据 |
|------|------|------|------|
| 主标题 | H1 | 固定文字 | "AI 驱动，一句话生成应用原型" |
| 副标题 | P | 固定文字 | "通过自然语言描述，快速生成完整的应用原型" |
| 开始创建 | 主按钮(白) | hover: 上浮 | 链接: /create |
| 查看演示 | 次按钮(透明) | hover: 背景变白 | 链接: /demo |

**动画效果**:
- 背景: `animation: float 20s ease-in-out infinite`
- 按钮: `transition: all 0.3s`

### 2.3 特性卡片

| 控件 | 包含 | 交互 |
|------|------|------|
| 特性图标 | Emoji/图标 | hover: 上浮 |
| 特性标题 | H3 | - |
| 特性描述 | P | - |

**数据结构**:
```javascript
features = [
  { icon: "🧠", title: "AI 智能理解", desc: "自然语言描述需求..." },
  { icon: "🔀", title: "流程图生成", desc: "自动生成业务流程图..." },
  { icon: "📱", title: "原型即所得", desc: "所见即所得的设计器..." },
  { icon: "📦", title: "一键导出", desc: "支持导出 PDF..." }
]
```

### 2.4 流程步骤

| 步骤 | 内容 | 交互 |
|------|------|------|
| 步骤1 | 描述需求 | 数字圆圈 + 标题 + 描述 |
| 箭头 | → | 固定 |
| 步骤2 | AI 分析 | 同上 |
| 箭头 | → | 固定 |
| 步骤3 | 原型生成 | 同上 |
