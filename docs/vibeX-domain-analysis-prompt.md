## 一、Domain Analysis Prompt（领域分析提示词）

```
# 角色
你是一位资深领域专家和系统架构师，精通 DDD（领域驱动设计）。

# 任务
请对以下业务领域进行深入分析：

## 输入信息
- 业务领域：[领域名称]
- 核心业务目标：[业务核心价值]
- 用户群体：[目标用户]
- 主要场景：[使用场景]

## 分析要求
1. 识别核心业务实体
2. 分析业务流程和关键路径
3. 确定业务边界和限界上下文
4. 识别核心域/支撑域/通用域

## 输出格式
请按照以下结构输出：
- 领域概述
- 核心实体清单
- 业务流程图
- 领域划分建议
```

## 五、实际拆解案例

### 电商系统领域拆解

| 子域 | 类型 | 说明 |
|------|------|------|
| 商品管理 | 核心域 | 商品上架、搜索、展示 |
| 订单处理 | 核心域 | 创建、支付、履约 |
| 支付 | 通用域 | 第三方支付集成 |
| 用户管理 | 通用域 | 注册、登录、权限 |
| 物流 | 支撑域 | 配送、跟踪 |
| 营销活动 | 支撑域 | 优惠券、促销 |

### AI 聊天应用领域拆解

| 子域 | 类型 | 说明 |
|------|------|------|
| 对话管理 | 核心域 | 多轮对话、上下文 |
| 用户意图识别 | 核心域 | NLP、意图分类 |
| 知识库 | 核心域 | RAG、知识检索 |
| 消息通道 | 通用域 | 多渠道接入 |
| 会员管理 | 通用域 | 用户体系、计费 |
| 敏感过滤 | 支撑域 | 内容安全 |

## 六、VibeX 领域拆解 Skill 输出格式

基于以上模板，VibeX 领域拆解 Agent 输出格式：

```json
{
  "domains": [
    {
      "name": "领域名称",
      "type": "core|supporting|generic",
      "priority": "P0|P1|P2",
      "description": "简要描述",
      "features": ["功能1", "功能2"],
      "entities": ["实体1", "实体2"],
      "dependencies": ["依赖领域"]
    }
  ],
  "mvpScope": {
    "core": ["必须实现的功能"],
    "extended": ["可后续添加"]
  }
}
```
