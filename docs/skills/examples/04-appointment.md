# VibeX 领域拆解示例

> 测试用例：预约挂号系统

**用户需求：** "我想做个医院预约挂号系统，患者可以预约医生"

---

## 拆解结果

### 总结
- 总领域数：7
- 核心域：3
- 支撑域：2
- 通用域：1
- 关联域：1

### 核心域

#### 1. 预约挂号 (core_1)
- **优先级**：P0
- **描述**：核心预约功能
- **子域**：
  - 科室管理（P0）：科室列表
  - 医生排班（P0）：出诊时间
  - 号源管理（P0）：号源数量、放号时间
  - 预约下单（P0）：选择医生、时间
  - 预约取消（P0）：取消、改约
- **实体**：[科室, 医生, 排班, 号源, 订单]

#### 2. 医院管理 (core_2)
- **优先级**：P0
- **描述**：医院基础信息
- **子域**：
  - 医院信息（P0）：名称、地址、简介
  - 科室信息（P0）：科室介绍、医生列表
  - 医生信息（P0）：姓名、职称、专长
- **实体**：[医院, 科室, 医生]

#### 3. 患者中心 (core_3)
- **优先级**：P0
- **描述**：患者用户体系
- **子域**：
  - 就诊人管理（P0）：添加、删除就诊人
  - 预约记录（P0）：历史预约
  - 档案管理（P1）：病历档案
- **实体**：[就诊人, 档案, 预约记录]

### 支撑域

#### 4. 叫号系统 (support_1)
- **优先级**：P1
- **描述**：现场排队
- **子域**：
  - 签到取号（P0）：现场取号
  - 队列管理（P0）：排队叫号
  - 候诊提醒（P1）：短信通知
- **实体**：[队列, 号票]

#### 5. 支付中心 (support_2)
- **优先级**：P1
- **描述**：费用支付
- **子域**：
  - 订单支付（P1）：挂号费支付
  - 退款处理（P1）：取消退款
- **实体**：[支付记录, 退款]

### 通用域

#### 6. 消息通知 (generic_1)
- **优先级**：P1
- **描述**：通知推送
- **子域**：
  - 预约成功通知（P0）
  - 就诊提醒（P0）
  - 停诊通知（P1）
- **实体**：[通知]

### 关联域

#### 7. 药品配送 (related_1)
- **关联方式**：扩展
- **发现原因**：处方外配、药店配送

---

### 依赖关系图

```
预约挂号(core_1) ← 依赖 ← 医院管理(core_2)
预约挂号(core_1) ← 依赖 ← 患者中心(core_3)
预约挂号(core_1) ← 依赖 ← 叫号系统(support_1)
预约挂号(core_1) ← 依赖 ← 支付中心(support_2)
```
