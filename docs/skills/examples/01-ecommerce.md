# VibeX 领域拆解示例

> 测试用例：电商系统

**用户需求：** "我想做个电商平台"

---

## 拆解结果

### 总结
- 总领域数：9
- 核心域：3
- 支撑域：3
- 通用域：2
- 关联域：1

### 核心域

#### 1. 商品中心 (core_1)
- **优先级**：P0
- **描述**：商品管理与展示
- **子域**：
  - 商品管理（P0）：上架、下架、编辑
  - 商品搜索（P0）：关键词搜索、分类筛选
  - 商品详情（P0）：SKU、库存、价格
  - 商品推荐（P1）：个性化推荐
- **功能**：[商品上架, 商品搜索, 商品详情, SKU管理, 库存管理, 价格管理]
- **实体**：[商品, SKU, 分类, 品牌]

#### 2. 交易中心 (core_2)
- **优先级**：P0
- **描述**：订单处理与支付
- **子域**：
  - 购物车（P0）：添加、删除、数量修改
  - 订单创建（P0）：提交订单、订单确认
  - 订单支付（P0）：支付宝、微信、银行卡
  - 订单状态（P0）：待付款、待发货、待收货、已完成
  - 售后退货（P1）：退货申请、退款
- **功能**：[购物车, 订单创建, 订单支付, 订单管理, 售后]
- **实体**：[订单, 购物车, 支付记录, 售后单]

#### 3. 用户中心 (core_3)
- **优先级**：P0
- **描述**：用户体系与会员
- **子域**：
  - 用户注册（P0）：手机号、邮箱、第三方登录
  - 用户登录（P0）：密码、验证码、Token
  - 会员体系（P1）：等级、积分、权益
  - 收货地址（P0）：新增、编辑、默认地址
- **功能**：[注册, 登录, 会员, 地址管理]
- **实体**：[用户, 地址, 会员, 积分记录]

### 支撑域

#### 4. 商家中心 (support_1)
- **优先级**：P1
- **描述**：商家管理后台
- **子域**：
  - 商家入驻（P1）：资质提交、审核
  - 店铺管理（P1）：店铺配置、装修
  - 商品管理（P1）：商家商品上传
  - 订单处理（P1）：发货、退款
- **功能**：[商家入驻, 店铺管理, 商品管理, 订单处理]
- **实体**：[商家, 店铺]

#### 5. 营销中心 (support_2)
- **优先级**：P1
- **描述**：营销活动
- **子域**：
  - 优惠券（P1）：发放、领取、使用
  - 促销活动（P1）：满减、折扣、秒杀
  - 拼团（P2）：拼团活动、拼团订单
  - 会员日（P2）：会员专享活动
- **功能**：[优惠券, 促销活动, 拼团, 会员日]
- **实体**：[优惠券, 活动, 拼团]

#### 6. 物流中心 (support_3)
- **优先级**：P1
- **描述**：物流配送
- **子域**：
  - 物流跟踪（P1）：运单生成、物流查询
  - 配送管理（P1）：发货、取消发货
  - 电子面单（P2）：面单生成、打印
- **功能**：[物流跟踪, 配送, 电子面单]
- **实体**：[运单, 物流信息]

### 通用域

#### 7. 消息通知 (generic_1)
- **优先级**：P2
- **描述**：消息推送
- **子域**：
  - 站内消息（P1）：订单通知、系统通知
  - 短信通知（P2）：验证码、订单短信
  - 邮件通知（P2）：订单邮件、订阅邮件
- **功能**：[站内消息, 短信, 邮件]
- **实体**：[消息, 通知模板]

#### 8. 内容运营 (generic_2)
- **优先级**：P2
- **描述**：内容管理
- **子域**：
  - Banner管理（P2）：首页轮播图
  - 文章管理（P2）：帮助中心、新闻资讯
  - 评价管理（P1）：商品评价、商家回复
- **功能**：[Banner, 文章, 评价]
- **实体**：[Banner, 文章, 评价]

### 关联域

#### 9. 直播带货 (related_1)
- **关联方式**：扩展
- **发现原因**：电商平台常见商业模式

---

### 依赖关系图

```
用户中心(core_3) ← 依赖 ← 商品中心(core_1)
用户中心(core_3) ← 依赖 ← 交易中心(core_2)
交易中心(core_2) ← 依赖 ← 物流中心(support_3)
交易中心(core_2) ← 依赖 ← 消息通知(generic_1)
```
