# VibeX 领域拆解示例

> 测试用例：AI 聊天机器人

**用户需求：** "我想做一个AI聊天助手，类似Character AI，可以创建角色进行对话"

---

## 拆解结果

### 总结
- 总领域数：7
- 核心域：3
- 支撑域：2
- 通用域：1
- 关联域：1

### 核心域

#### 1. 对话管理 (core_1)
- **优先级**：P0
- **描述**：核心对话能力
- **子域**：
  - 多轮对话（P0）：上下文记忆
  - 对话生成（P0）：AI 回复生成
  - 角色设定（P0）：人格、背景、语气
  - 对话历史（P0）：存储、搜索
- **实体**：[对话, 消息, 角色]

#### 2. 角色创建 (core_2)
- **优先级**：P0
- **描述**：AI 角色定制
- **子域**：
  - 角色模板（P0）：预设角色库
  - 自定义角色（P0）：姓名、头像、人设
  - Prompt工程（P0）：系统提示词优化
  - 角色市场（P1）：分享、下载
- **实体**：[角色, 模板, Prompt]

#### 3. 用户交互 (core_3)
- **优先级**：P0
- **描述**：用户界面与体验
- **子域**：
  - 对话界面（P0）：Web/App
  - 语音对话（P1）：TTS、ASR
  - 表情反馈（P1）：点赞、不喜欢
- **实体**：[用户, 反馈]

### 支撑域

#### 4. RAG 知识库 (support_1)
- **优先级**：P1
- **描述**：知识增强
- **子域**：
  - 文档上传（P1）：PDF、txt
  - 向量检索（P1）：语义搜索
  - 知识引用（P1）：来源标注
- **实体**：[知识库, 文档, 向量]

#### 5. 敏感过滤 (support_2)
- **优先级**：P0
- **描述**：内容安全
- **子域**：
  - 关键词过滤（P0）：敏感词库
  - 内容审核（P0）：AI 审核
  - 举报处理（P1）：用户反馈
- **实体**：[敏感词, 审核记录]

### 通用域

#### 6. 消息通道 (generic_1)
- **优先级**：P1
- **描述**：多渠道接入
- **子域**：
  - Web端（P0）
  - App端（P1）
  - API接入（P2）
- **实体**：[通道, 接入]

### 关联域

#### 7. 虚拟陪伴 (related_1)
- **关联方式**：扩展
- **发现原因**：情感计算、语音通话

---

### 依赖关系图

```
对话管理(core_1) ← 依赖 ← 敏感过滤(support_2)
对话管理(core_1) ← 依赖 ← RAG知识库(support_1)
角色创建(core_2) ← 依赖 ← 对话管理(core_1)
```
